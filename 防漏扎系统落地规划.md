# 防漏扎布线板检测平台 CV 落地全方位规划

本项目旨在从零开始，为您打造一个**极致性价比、效果最优**的计算机视觉（CV）防漏扎检测系统。系统将通过摄像头自动识别布线板上的扎带，并与 Excel 产品清单中的标准数量进行比对，实现漏扎自动报警。

## 1. 硬件选型规划 (主打：最便宜的硬件 + 最好的效果)

在工业视觉中，“最好的效果”往往**七分靠打光和成像，三分靠算法**。在计算端我们可以采用高性价比的边缘计算方案。

### 极致性价比方案 (总成本预估：约 ¥1000 - ¥1500)
- **计算平台（推理端）**：**RK3588 开发板（如香橙派 Orange Pi 5）** 或 **Intel N100 迷你主机**。
  - *理由*：自带 NPU / OpenVINO 加速，跑轻量级目标检测模型（如 YOLOv8n）完全满足实时检测需求。
- **相机**：**普通工业 USB 摄像头** (由于清单表明存在1.5米以上的布线板，可能需要 **2个** 摄像头拼接覆盖)。
- **光源**：**LED 环形光源 / 条形光源**（极其重要，能消除车间环境光变化）。

## 2. 软件与算法架构 (主打：前沿开源，效果最好)

### 核心视觉算法：YOLOv8 或 YOLOv10 (Nano/Small 版本)
- **任务定义**：训练模型识别并框出图片中所有的“扎带”。

### ⚠️ 针对《产品清单》的定制化软件需求 (本次更新)
根据对 `产品清单.xlsx` 的深度解析，我们发现清单包含 `1米5一下布线板（直式）` 和 `1米5以上（U型弯）` 等类型，且分为“互锁部件”和“功率部件”，单板扎带总数高达 **60个** (14+17+11+18)。因此，软件业务逻辑必须满足以下定制化需求：

1. **多型号一键切换与配置读取**：
   - 系统界面需提供型号切换功能（或通过条码枪扫码）。
   - 切换后，Python 后端自动解析 Excel，提取当前型号下“互锁部件”和“功率部件”各自要求的**标准扎带数量**。
2. **多类别目标检测 (多分类 YOLO)**：
   - 清单中对扎带有具体区分，如“普通扎带”和“自动裁剪功能扎带”。
   - 如果这两类扎带**在外观上有明显区别**，我们在标注数据时，就需要把它们标为不同的类别（类0：普通扎带，类1：自动裁剪扎带），让 AI 分别统计数量。
3. **超大画幅与小目标检测策略**：
   - **痛点**：对于“1.5米以上”的布线板，如果要看清板子上多达 60 个微小的扎带，单摄像头绝对会因为分辨率不足而导致严重漏检。
   - **软件策略**：软件必须支持**多路视频流并发推理**或**图像滑动窗口推理 (SAHI 切片检测)** 技术。即把超大分辨率的图像切成小块送给模型识别，最后再汇总总数，确保对小目标 100% 的召回率。

## 3. 业务闭环与落地路线图

1. **第一阶段：硬件铺设**
   - 购买多路工业相机与光源，在 1.5 米工作台上安装好拍摄矩阵。
2. **第二阶段：实境采图与标注**
   - 按照 Excel 中列出的各个型号（直式、U型弯），每种型号拍摄不同光线下的照片，累计 300-500 张。
   - 使用 AnyLabeling 对“互锁部件扎带”和“功率部件扎带”进行精准框选。
3. **第三阶段：软件原型与逻辑打通**
   - 编写界面的“防漏扎比对逻辑”：`如果 (AI 识别总数 < Excel 提取总数) -> 触发红色漏扎警报并蜂鸣`。
4. **第四阶段：现场联调**
   - 部署到 RK3588/N100 硬件，跑通端到端流程，针对错检漏检持续优化。
